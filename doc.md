# Техническая документация проекта

## Назначение

`monolit` отправляет личные VK-напоминания о передаче показаний счетчиков,
оплате коммунальных услуг, пользовательских событиях и доступности настроенного
узла, ведет список покупок в личных сообщениях VK и присылает новости Cherinfo.

## Архитектура

- `MonthlyReminderScheduler` запускает обычные ежемесячные напоминания, создание
  записей нового месяца и проверку истекших интервальных отсрочек.
- `MonthlyReminderService` управляет состоянием ежемесячных напоминаний и
  формирует сообщения с VK-клавиатурой.
- Callback handlers разбирают действия выполнения и отсрочки из VK payload.
- `MonthlyReminderRepository` сохраняет состояние в PostgreSQL.
- `ReminderConversationService` управляет пошаговым созданием одноразовых
  пользовательских напоминаний через постоянную VK-клавиатуру.
- `CustomReminderService` отправляет пользовательские напоминания, список,
  карточки и обрабатывает inline-действия.
- `BotMainMenuService` централизует постоянную клавиатуру главного меню.
- `ShoppingListConversationService` управляет пошаговым добавлением покупок.
- `ShoppingListService` хранит покупки, показывает список, переключает статус
  элементов и очищает купленные покупки.
- `CherinfoNewsScheduler` запускает первичную и hourly-проверку новостей
  Cherinfo.
- `CherinfoNewsService` выбирает первичные или новые публикации, отправляет
  полный текст и временные вложения изображений в VK и обновляет компактное
  состояние обработки новостной ленты.
- `NewsSourceClient` задает общий контракт новостного источника: получение
  списка публикаций и деталей отдельной новости.
- `CherinfoNewsHttpClient` получает RSS `https://cherinfo.ru/rss/news/` для
  списка публикаций и загружает страницы отдельных новостей для полного текста.
- `CherinfoNewsParser` разбирает RSS Cherinfo, текст публикации, одиночные
  изображения и галереи статьи с настраиваемыми Cherinfo-путями качества
  изображений.
- `CherinfoNewsImageDownloader` скачивает изображения новостей во временные
  файлы для загрузки во VK.

## Публичные контракты

- Ежемесячное напоминание содержит отдельный ряд с кнопкой выполнения и ряд из
  кнопок `10 минут`, `1 час`, `3 часа`, `12 часов`,
  `Не напоминать сегодня`.
- Callback payload имеет envelope с `type`, `version` и `data`.
- Типы callback: `monthly_reminder_done` и `monthly_reminder_postpone`, версия
  `1`.
- Таблица `monthly_reminder` дополнительно хранит `postponed_until` и
  `postponement_type`.
- Постоянная клавиатура главного меню содержит группы `Напоминания` и
  `Покупки`.
- Группа `Напоминания` открывает подменю с действиями `Новое напоминание`,
  `Мои напоминания` и возвратом `Главное меню`.
- Группа `Покупки` открывает подменю с действиями `Список покупок`,
  `Добавить покупку` и возвратом `Главное меню`.
- Пользовательское напоминание создается пошагово с быстрым выбором даты и
  времени. Полный формат даты и времени не требуется.
- Список покупок отображается постранично по пять элементов.
- Команда `Весь список покупок` отправляет все покупки одним сообщением только
  для просмотра, без inline-действий изменения данных.
- Inline-кнопки списка покупок используют компактные подписи `←`, `→`, `➕`,
  `🧹`, `✖` и `↩`, чтобы служебные действия помещались на телефоне.
- Callback payload для списка покупок имеет type `shopping_list_action` и
  version `1`.
- Новости Cherinfo отправляются личными сообщениями VK с заголовком,
  временем/датой публикации и полным текстом новости без ссылки на публикацию.
- К новости Cherinfo прикладываются распознанные изображения публикации, но не
  больше 10 изображений на одну новость.
- Изображения Cherinfo дедуплицируются; для галерей используются ссылки на
  крупные изображения, когда сайт предоставляет их в разметке.
- Таблица `cherinfo_news_state` хранит одну запись состояния Cherinfo:
  checkpoint последней верхней новости ленты и ограниченный список последних
  отправленных URL для защиты от дублей.

## Важные бизнес-правила

- Интервальная отсрочка блокирует обычный scheduler и отправляется отдельным
  минутным scheduler после наступления срока.
- `Не напоминать сегодня` блокирует отправки до следующего календарного дня в
  зоне `monolit.reminders.zone`, затем действуют обычные cron-окна.
- Последнее действие отсрочки заменяет предыдущее.
- Выполнение напоминания очищает активную отсрочку.
- При запуске отсутствующие записи текущего месяца создаются автоматически.
- Пользовательские напоминания одноразовые и автоматически не повторяются.
- Просроченные во время простоя пользовательские напоминания помечаются
  пропущенными и после запуска не отправляются.
- Пользовательские напоминания проверяются каждую секунду и не отправляются
  раньше выбранного времени.
- Входящие команды и действия принимаются только от пользователя `VK_MY_ID`.
- Список содержит до пяти ближайших активных напоминаний на странице.
- Навигация списка и действия карточки отправляют новые VK-сообщения. Это
  исключает нестабильное редактирование сообщений через используемый VK SDK.
- Список покупок хранит дубликаты как отдельные элементы.
- В списке покупок элементы отображаются в стабильном порядке добавления, чтобы
  номерные кнопки соответствовали видимым номерам после переключения статуса.
- Номерные кнопки списка покупок разбиваются по рядам максимум по пять кнопок,
  чтобы inline-клавиатура оставалась валидной для VK.
- Страница списка покупок содержит максимум пять товаров, чтобы вместе с
  навигацией и служебными действиями inline-клавиатура не превышала лимит VK на
  общее количество кнопок.
- Непокупленные товары показываются без статусного маркера, купленные товары
  показываются с `✅` после названия.
- В полном просмотре списка покупок используется тот же статусный формат, что и
  в обычном списке: купленные товары показываются с `✅`, непокупленные товары
  без статусного маркера.
- Добавление покупок сохраняет элементы только после нажатия `Готово`; `Отмена`
  удаляет черновик без сохранения.
- Активный диалог создания напоминания имеет приоритет над диалогом списка
  покупок и навигацией постоянного меню. Активный диалог добавления покупок
  имеет приоритет над командами главного меню и подменю до `Готово` или
  `Отмена`.
- При первом запуске функциональности с пустым состоянием Cherinfo
  отправляются последние пять новостей Cherinfo.
- Повторные проверки Cherinfo выполняются каждый час и отправляют только
  публикации, появившиеся выше сохраненного checkpoint новостной ленты.
- Список публикаций Cherinfo берется из RSS `https://cherinfo.ru/rss/news/`;
  HTML-страница `https://cherinfo.ru/news` не используется как источник ленты.
- Пути качества изображений Cherinfo задаются свойствами
  `monolit.news.cherinfo.image.big-path`,
  `monolit.news.cherinfo.image.medium-path` и
  `monolit.news.cherinfo.image.small-path`.
- Несколько новых новостей Cherinfo отправляются от более старой к более новой
  среди найденных новых публикаций.
- Для отправки полной новости загружается страница публикации. Если полный текст
  не распознан, новость не отправляется и не помечается отправленной.
- Картинки Cherinfo не сохраняются постоянно. Допустимы только временные файлы
  на время загрузки во VK; после попытки отправки они удаляются.
- Если картинки не найдены или не скачались, полный текст новости отправляется
  без вложений.
- Ошибка получения или разбора ленты, страницы публикации или картинки Cherinfo
  логируется и не должна останавливать приложение.

## Технические решения

- Решение: время вычисляется через Spring bean `Clock`.
  Причина: единая настроенная зона и детерминированные тесты.
  Последствия: календарные границы и интервалы не зависят от системной зоны JVM.
- Решение: `spring.jpa.hibernate.ddl-auto=update`.
  Причина: поддержка запуска с пустой локальной PostgreSQL и обновления
  существующей таблицы.
  Последствия: Hibernate автоматически изменяет схему при запуске.
- Решение: новость Cherinfo считается отправленной по абсолютной ссылке на
  публикацию, а граница новых публикаций определяется checkpoint верхней новости
  ленты.
  Причина: ссылка является стабильным идентификатором публикации в ленте.
  Последствия: повторный запуск приложения не отправляет ранее сохраненные
  новости повторно и не досылает старые публикации за пределами первичного окна
  из пяти новостей.
- Решение: вместо истории всех отправленных новостей Cherinfo хранится одна
  строка состояния с checkpoint и ограниченным списком последних 50 URL.
  Причина: для hourly-проверки отсортированной ленты не требуется постоянно
  накапливать все когда-либо отправленные новости.
  Последствия: использование БД остается небольшим и предсказуемым.
- Решение: ссылка Cherinfo используется только как внутренний идентификатор и не
  включается в VK-сообщение.
  Причина: пользователь должен видеть полный текст новости без необходимости
  переходить по ссылке.
  Последствия: для каждой новой публикации требуется загрузка страницы новости и
  парсинг полного текста.
- Решение: список новостей Cherinfo берется из RSS, а детали публикации
  загружаются отдельно со страницы новости.
  Причина: RSS является стабильным машинным контрактом для ленты и уже содержит
  заголовок, ссылку и дату публикации.
  Последствия: парсинг списка не зависит от HTML-разметки страницы ленты; HTML
  остается нужен только для полного текста и изображений статьи.
- Решение: сервис публикации зависит от `NewsSourceClient` и общих DTO
  `NewsItem`/`NewsDetails`, а Cherinfo-специфичные правила находятся в
  `CherinfoNewsHttpClient` и `CherinfoNewsParser`.
  Причина: будущие источники новостей должны подключаться через общий контракт
  без копирования логики отправки в VK.
  Последствия: добавление нового источника потребует отдельной реализации
  клиента/парсера и настройки состояния, но не переписывания базового контракта
  новости.
- Решение: при извлечении изображений Cherinfo приоритет имеет галерея статьи,
  затем изображения из текста статьи, затем `og:image` как fallback.
  Причина: `og:image` и превью галереи могут дублировать основную картинку или
  указывать на изображение меньшего качества.
  Последствия: одиночная картинка не отправляется дважды, а галерея отправляет
  доступные крупные изображения в пределах лимита вложений.

## Сборка и проверка

- Требуются Java 21, Maven и PostgreSQL.
- Локальные параметры подключения находятся в `application.properties`.
- LongPoll можно отключить для безопасного локального запуска свойством
  `monolit.vk.long-poll.enabled=false`.
- Для полного прогона используется `mvn clean verify`.
