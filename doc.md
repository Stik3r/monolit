# Техническая документация проекта

## Назначение

`monolit` отправляет личные VK-напоминания о передаче показаний счетчиков,
оплате коммунальных услуг, пользовательских событиях и доступности настроенного
узла.

## Архитектура

- `MonthlyReminderScheduler` запускает обычные ежемесячные напоминания, создание
  записей нового месяца и проверку истекших интервальных отсрочек.
- `MonthlyReminderService` управляет состоянием ежемесячных напоминаний и
  формирует сообщения с VK-клавиатурой.
- Callback handlers разбирают действия выполнения и отсрочки из VK payload.
- `MonthlyReminderRepository` сохраняет состояние в PostgreSQL.
- `ReminderConversationService` управляет пошаговым созданием одноразовых
  пользовательских напоминаний через постоянную VK-клавиатуру.
- `CustomReminderService` отправляет пользовательские напоминания, список,
  карточки и обрабатывает inline-действия.

## Публичные контракты

- Ежемесячное напоминание содержит отдельный ряд с кнопкой выполнения и ряд из
  кнопок `10 минут`, `1 час`, `3 часа`, `12 часов`,
  `Не напоминать сегодня`.
- Callback payload имеет envelope с `type`, `version` и `data`.
- Типы callback: `monthly_reminder_done` и `monthly_reminder_postpone`, версия
  `1`.
- Таблица `monthly_reminder` дополнительно хранит `postponed_until` и
  `postponement_type`.
- Постоянная клавиатура содержит `Новое напоминание` и `Мои напоминания`.
- Пользовательское напоминание создается пошагово с быстрым выбором даты и
  времени. Полный формат даты и времени не требуется.

## Важные бизнес-правила

- Интервальная отсрочка блокирует обычный scheduler и отправляется отдельным
  минутным scheduler после наступления срока.
- `Не напоминать сегодня` блокирует отправки до следующего календарного дня в
  зоне `monolit.reminders.zone`, затем действуют обычные cron-окна.
- Последнее действие отсрочки заменяет предыдущее.
- Выполнение напоминания очищает активную отсрочку.
- При запуске отсутствующие записи текущего месяца создаются автоматически.
- Пользовательские напоминания одноразовые и автоматически не повторяются.
- Просроченные во время простоя пользовательские напоминания помечаются
  пропущенными и после запуска не отправляются.
- Пользовательские напоминания проверяются каждую секунду и не отправляются
  раньше выбранного времени.
- Входящие команды и действия принимаются только от пользователя `VK_MY_ID`.
- Список содержит до пяти ближайших активных напоминаний на странице.
- Навигация списка и действия карточки отправляют новые VK-сообщения. Это
  исключает нестабильное редактирование сообщений через используемый VK SDK.

## Технические решения

- Решение: время вычисляется через Spring bean `Clock`.
  Причина: единая настроенная зона и детерминированные тесты.
  Последствия: календарные границы и интервалы не зависят от системной зоны JVM.
- Решение: `spring.jpa.hibernate.ddl-auto=update`.
  Причина: поддержка запуска с пустой локальной PostgreSQL и обновления
  существующей таблицы.
  Последствия: Hibernate автоматически изменяет схему при запуске.

## Сборка и проверка

- Требуются Java 21, Maven и PostgreSQL.
- Локальные параметры подключения находятся в `application.properties`.
- LongPoll можно отключить для безопасного локального запуска свойством
  `monolit.vk.long-poll.enabled=false`.
- Для полного прогона используется `mvn clean verify`.
